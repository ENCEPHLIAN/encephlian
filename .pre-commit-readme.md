# Pre-commit hooks: secret-leak defense

Structural defense against committing JWTs, API keys, service-role tokens, and
private keys. Configured in `.pre-commit-config.yaml`. Lives in this repo and in
`encephlian-core` with identical setup.

## One-time setup (per contributor, per machine)

```sh
pip install pre-commit
pre-commit install
```

That's it. `pre-commit install` writes `.git/hooks/pre-commit` which invokes the
framework. Every `git commit` from now on scans staged files; the commit is
blocked if any hook fails.

If you've never used pre-commit before in this repo, you also need to seed the
detect-secrets baseline (one-time, repo-wide):

```sh
pip install detect-secrets
detect-secrets scan \
    --exclude-files 'package-lock\.json|bun\.lock|node_modules/.*|dist/.*' \
    > .secrets.baseline
git add .secrets.baseline
```

This file records existing "known" findings so the hook doesn't flag them on
every commit. New findings (anything not in the baseline) will fail the hook.

## What it catches

| Hook                              | Catches                                                                  |
| --------------------------------- | ------------------------------------------------------------------------ |
| `detect-secrets`                  | high-entropy strings, AWS keys, Slack tokens, Stripe keys, JWTs, etc.    |
| `detect-private-key`              | PEM/SSH/OpenSSL private-key blocks (`BEGIN RSA PRIVATE KEY`, etc.)       |
| `encephlian-secret-patterns`      | Custom regex: any JWT (`eyJhbGci...`), literal `SUPABASE_*_KEY = "eyJ"`, Azure storage account keys (88-char base64) |
| `check-added-large-files`         | >2 MB blobs (often model weights or credential dumps)                    |

The custom regex layer exists because detect-secrets's JWT plugin can miss
novel issuers, and the literal-Supabase-key shape is the exact regression we
scrubbed on 2026-06-02.

## What it WON'T catch

- **Already-committed history.** Hooks fire on new commits only. To purge
  history, use `git-filter-repo` and then rotate the leaked credential.
- **Plain-text credentials with no recognizable shape.** A 32-char hex API key
  with no surrounding context may slip past detect-secrets if its entropy
  score is below threshold. Always namespace secrets with their provider name
  (e.g. `STRIPE_SECRET_KEY=...`) so future hooks have a hook point.
- **JWTs with non-standard headers.** Our regex anchors on `eyJhbGci` (the
  base64 of `{"alg":`). A token whose first claim is something other than `alg`
  (rare but legal) would not match the custom hook; detect-secrets's JWT
  plugin should still catch it.
- **Credentials that arrive only via `.env` files** — those should never be
  staged in the first place; `.env*` is in `.gitignore`.

## Adding an exception

Almost never needed. In order of preference:

1. **Move the secret to an env var.** Source of truth lives in
   Vercel/Azure/Supabase project secrets, not source.
2. **Add a per-line allow-comment.** Append ` # pragma: allowlist secret` to
   the line. Works for both detect-secrets and the custom hooks.
3. **Add to the baseline.** For false positives (test fixtures, doc examples),
   run `detect-secrets scan --baseline .secrets.baseline` and commit the
   updated baseline. Review the diff before committing.

## Running ad-hoc

```sh
pre-commit run --all-files           # scan the whole repo
pre-commit run --files path/to/x.py  # scan specific files
pre-commit autoupdate                # bump hook revs in .pre-commit-config.yaml
```

## If you trip a hook

1. Read the error message; it names the file, line, and pattern.
2. Remove the secret from the file. If it was ever rendered into a shell or
   transcript, treat it as compromised:
   - Rotate the credential in the provider console FIRST.
   - Scrub from history if the file was previously tracked:
     `git filter-repo --path path/to/file --invert-paths`.
3. Re-stage and re-commit.

## If a hook is broken / blocking a legitimate commit

`git commit --no-verify` bypasses every hook. Use sparingly and only after
manually inspecting the diff for secrets. If you find a real bug in the hook,
fix it in `scripts/check_secrets.py` or `.pre-commit-config.yaml`.
